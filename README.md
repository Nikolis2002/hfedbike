# Hierarchical Federated Learning for Bike-Sharing Demand

Hour-ahead demand forecasting for the New York City Citi Bike network,
trained with a hybrid federated-learning architecture: four geographic
neighborhoods, static leaders, peer-to-peer ring reduce inside each
neighborhood, and a non-computational router between leaders.

## Quickstart

Prerequisites: Docker, and an external `my_macvlan` network on the host
(one-time setup — replace `eth0` with your active interface):

```bash
docker network create -d macvlan \
  --subnet=192.168.1.0/24 --gateway=192.168.1.1 \
  -o parent=eth0 my_macvlan
```

### Run the federation

```bash
# 1. Build the image (once, or after requirements.txt changes)
docker build -t federated-node .

# 2. Clear previous results if you want a clean run:
docker run --rm -v "$(pwd)/results/current:/r" alpine rm -f /r/*.csv

# 3. Launch 16 nodes + the coordinator
NODE_SCRIPT=v2_node.py docker compose up -d

# 4. Wait ~20 minutes; follow progress with
docker logs -f nodeX

# 5. When all nodes are in Exited(0), regenerate plots and summary stats:
python3 analysis/test_for_year.py
```

Per-node CSVs land in `results/current/<NODE>_results.csv` with columns
`week, base_mae, base_mse, freeze_mae, freeze_mse, fed_mae, fed_mse`.
`analysis/test_for_year.py` aggregates them, writes
`figures/mae_comparison.{pdf,png}`, and prints the weekly-average MAE for
each model plus pairwise Wilcoxon signed-rank p-values.

### Regenerate the baseline model (optional)

```bash
# Grid search over ~1,700 configurations (hours of CPU time)
python3 baseline/model_search/find_the_best_baseline.py

# Retrain the winning configuration on the full 2023 data
python3 baseline/model_search/best_baseline_model.py
# Writes data/2024/_model_two_layers.keras (overwrite the in-tree baseline)
```

## Headline results

On the 52 consecutive ISO weeks of 2024, across 16 subzones:

| Model                  | Avg weekly MAE (z-score units) | 95% CI          |
|------------------------|-------------------------------:|-----------------|
| Frozen baseline        | 0.648                          | [0.635, 0.661]  |
| Bias-adapted baseline  | 0.571                          | [0.561, 0.582]  |
| **Federated**          | **0.459**                      | [0.446, 0.472]  |

Federated vs. frozen: **−29.2%**. Federated vs. bias-adapted: **−19.6%**.
All three pairwise differences significant at `p < 1e-8`
(paired Wilcoxon).

## Data notes

- Raw 2023 CSVs in `data/2023/` were downloaded from
  <https://citibikenyc.com/system-data>; weather from OpenWeatherMap.
- Per-subzone CSVs in `data/2024/entire_year/` are the output of
  `baseline/data_preparation/make_neighbs_2024/make_the_final_csvs.py`
  (station → k-means subzone mapping, hourly aggregation).
- The frozen baseline keras model lives at
  `data/2024/_model_two_layers.keras`; both `federated/p2p_node/model.py`
  and the docker-compose mount expect it there.
